## 프론트엔드 (LLMate_FE) 업데이트 내역

### InputPage2 (자기소개서 입력 페이지)

1.  **UI/UX 개편**:
    *   `inputpage2.css`를 통해 전체적인 디자인 및 레이아웃을 현대적으로 개선했습니다.

2.  **기능 개선**:
    *   **질문 동적 추가**: 여러 개의 자기소개서 문항을 동적으로 추가하고 입력할 수 있도록 기능을 확장했습니다.
    *   **선택형 경험 관리**: 기존의 단일 텍스트 입력 방식에서, 미리 저장된 여러 경험 중 자기소개서에 포함할 항목을 사용자가 직접 선택하는 방식으로 변경했습니다.

3.  **API 연동 수정**:
    *   여러 문항 정보를 배열 형태로 백엔드에 전송하도록 API 요청 데이터 구조를 변경했습니다.
    *   요청 주소를 신규 백엔드 엔드포인트인 `/essays/generate/full2`로 변경했습니다.

### OutputPage2 (결과 확인 페이지)

1.  **UI/UX 개편**:
    *   **다중 결과 조회**: 백엔드로부터 받은 여러 개의 자기소개서 결과를 한 페이지에서 모두 확인할 수 있도록 구조를 변경했습니다.
    *   **탐색 사이드바**: 화면 왼쪽에 접고 펼 수 있는 사이드바를 추가하여, 사용자가 문항 목록을 보고 쉽게 다른 결과물로 이동할 수 있도록 개선했습니다.

2.  **기능 수정 및 버그 픽스**:
    *   **수정 기능 복원**: 페이지 이동 없이 현재 페이지에서 바로 수정 요청을 보낼 수 있도록 기능을 다시 구현했습니다.
    *   **데이터 매핑 오류 수정**: 백엔드에서 받은 데이터의 속성명(`feedback_text`, `llm_model`)과 프론트엔드에서 사용하려던 속성명(`text`, `reviewer`)이 달라 발생했던 렌더링 오류(`TypeError`)를 해결했습니다.
    *   **로고 이미지 오류 수정**: 모델 ID(e.g., `gpt-4o-mini`)를 실제 로고 이미지와 이름(e.g., `ChatGPT`)에 맞게 변환하는 로직을 추가하여 이미지가 정상적으로 표시되도록 수정했습니다.
